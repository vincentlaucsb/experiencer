# Git & Documentation

## Commit Messages
- Use conventional commits format
- Examples: `feat:`, `fix:`, `refactor:`, `docs:`
- Keep first line under 72 characters
- Add detailed description if needed

## Branch Strategy
- Main branch: `master` or `main`
- Feature branches: `feature/description`
- Bug fixes: `fix/description`

## Code Comments
- Use JSDoc for public APIs
- Explain "why" not "what"
- Keep comments up to date with code
- Remove commented-out code

## Documentation Updates
- Update README.md when adding major features
- Include migration notes for breaking changes in docs/MIGRATION.md
- Keep setup instructions current in README.md
- Update docs/TODO.md with new planned work
